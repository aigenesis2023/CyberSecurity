Accessibility Statement

Product: Cybersecurity Awareness Training
Version: 1.0
Statement date: March 2026

This training application is designed to conform to Web Content Accessibility Guidelines (WCAG) 2.1 Level AA. We are committed to ensuring that the learning experience is accessible to the widest possible audience, including users of assistive technology.

Conformance Status

This product is substantially conformant with WCAG 2.1 Level AA. The known exceptions and partial conformances are documented below.

Measures Taken

  • All text meets a minimum 4.5:1 contrast ratio against its background.
  • All non-text UI components (progress bars, segment indicators, borders) meet a 3:1 contrast ratio.
  • All interactive elements are keyboard operable (Tab, Enter, Space, Escape).
  • Focus indicators are visible on all interactive elements.
  • Modal dialogs trap focus and return focus to the triggering element on close.
  • All decorative images and icons are hidden from assistive technology (aria-hidden="true").
  • Appropriate ARIA roles, labels, and live regions are used throughout.
  • A skip navigation link is provided.
  • Animations and transitions are suppressed when the user has enabled prefers-reduced-motion.
  • Touch targets are a minimum of 44×44 CSS pixels on mobile viewports.
  • Page titles are updated on each screen transition.
  • The lang attribute is set to en on the root element.

Known Exceptions and Limitations

Success CriterionLevelStatusNotes
2.2.1 Timing Adjustable A Exception applied Mission 03 (Prompt Injection) includes a countdown timer that is essential to the learning objective. The timed constraint simulates real-world conditions under which prompt injection attacks must be identified. Removing, pausing, or extending the time limit would invalidate the activity. This exception is permitted under WCAG 2.2.1 (essential timing). Note: the timer is paused automatically during all feedback interactions so users are not penalised while reading instructional content.
1.4.1 Use of Color A Partial The integrity score indicator changes colour (green → amber → red) to signal severity. A numeric percentage value is displayed alongside the colour at all times, providing a non-colour alternative.
1.4.10 Reflow AA Partial The simulated document and chat interfaces within missions may require horizontal scrolling at viewport widths below 375px. The application is designed and tested for a minimum viewport width of 375px (smallest mainstream mobile device).

Testing

This application has been evaluated using:

  • Manual keyboard-only navigation testing
  • Colour contrast verification against WCAG 2.1 AA ratios
  • Code review against WCAG 2.1 Level AA success criteria
